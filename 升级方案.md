## 一、项目现状诊断

### 1.1 当前问题清单

#### 严重问题（阻塞上线）
| # | 问题 | 位置 | 影响 |
|---|------|------|------|
| 1 | **首页被模板页覆盖** | `src/app/page.tsx` 是 Next.js 默认模板 | 访问 `/` 显示 "To get started..." 而非活动首页 |

| 7 | **后台无认证保护** | `/admin/*` 路由 | 任何人无需登录即可访问后台 |
| 8 | **后台全是静态页面** | 10个 admin 页面 | 按钮点击无任何实际功能 |

#### 代码结构问题
| # | 问题 | 说明 |

| 4 | 类型定义冗余 | `src/types/index.ts` 与 Prisma schema 重复维护 |
| 5 | 环境变量不规范 | 没有 `.env` 默认值文件 |

## 二、升级目标

### 2.1 核心目标
1. **完全匹配 Excel 策划案**：活动环节、时间规划、队伍规模、游戏站设置、轮转机制、寻宝赛全部按策划案实现
2. **全功能后端对接**：所有页面从 mock 数据改为真实数据库 CRUD
3. **代码质量重构**：清理混乱代码，建立规范的组件体系和架构
4. **角色权限系统**：老师、活动管理员、工作人员、学生四种角色，各有不同界面和权限

### 2.2 技术选型
| 项目 | 选择 | 说明 |
|------|------|------|
| 前端框架 | Next.js 15 + React 19 + TypeScript | 保持现有 |
| 样式 | Tailwind CSS v4 | 保持现有 |
| 动画 | Framer Motion | 保持现有 |
| 图标 | Lucide React | 保持现有 |
| 表单 | React Hook Form + Zod | 保持现有 |
| ORM | Prisma | 保持现有 |
| 数据库 | **PostgreSQL（Supabase）** | 从占位符改为 Supabase 免费实例 |
| 认证 | NextAuth v5 | 保持现有，扩展角色系统 |
| 部署 | **Vercel（免费）** | 前端+后端一体部署 |
| 地图 | 在线地图（高德/腾讯地图嵌入） | 寻宝区域展示 |

### 2.3 为什么选 Supabase + Vercel
- **零成本**：Supabase 免费版 500MB 数据库 + 5万月活，Vercel 免费版足够
- **200并发无压力**：Supabase 是真正的 PostgreSQL，连接池自带，200人同时在线完全没问题
- **自带管理界面**：Supabase Dashboard 可以直接查看/编辑数据，方便活动当天应急
- **自动备份**：Supabase 自动备份，数据安全有保障
- **部署简单**：Vercel 一键部署，连接 GitHub 仓库自动构建
- **扩展性好**：如果以后用户量增加，Supabase 升级计划即可

## 三、角色与权限设计

### 3.1 角色定义

```
Role 枚举（系统权限角色）：
  TEACHER          # 老师（团委书记/副书记）
  ADMIN            # 活动管理员（学生会组织部等）
  STAFF            # 工作人员（裁判、巡查、签到等）
  STUDENT          # 学生（包含普通队员和队长，队长通过 TeamMember.isCaptain 标识）

StudentPosition 枚举（学生班委职务，支持多选+主职务）：
  CLASS_MONITOR    # 班长
  LEAGUE_SECRETARY # 团支书
  STUDY_COMMISSAR  # 学习委员
  LIFE_COMMISSAR   # 生活委员
  CULTURE_COMMISSAR # 文体委员
  OTHER            # 其他班委

Permission 枚举（管理员权限细粒度控制）：
  MANAGE_REGISTRATIONS   # 管理报名
  MANAGE_TEAMS           # 管理分组
  MANAGE_STAFF           # 管理工作人员
  MANAGE_SCHEDULE        # 管理日程
  MANAGE_ANNOUNCEMENTS   # 管理公告
  MANAGE_SCORES          # 管理积分
  MANAGE_MATERIALS       # 管理物资
  MANAGE_ROTATION        # 管理轮转排班
  MANAGE_TREASURE        # 管理寻宝
  VIEW_FEEDBACKS         # 查看反馈
  MANAGE_SETTINGS        # 系统设置
  VIEW_LOGS              # 查看活动日志
  EXPORT_DATA            # 导出数据
```

**队长识别逻辑（处理兼职问题）**：
- 学生报名时选择**主职务**（必填）+ **兼任职务**（可选，可多选）
- 自动分组时，系统按**主职务**优先级识别队长候选人：
  1. 主职务为"班长"的学生优先
  2. 如果某队没有班长，则选主职务为"团支书"的学生
  3. 依次类推：学习委员 → 生活委员 → 文体委员 → 其他
- 管理员可手动调整队长

**管理员权限分配**：
- 管理员账号可分配细粒度权限（Permission）
- 例如：某管理员只负责积分录入，只分配 `MANAGE_SCORES` 权限
- 拥有 `MANAGE_SETTINGS` 权限的管理员可以管理其他管理员的权限
- 老师角色默认拥有所有权限（不可修改）
- 超级管理员（seed 预设的第一个账号）拥有所有权限且不可被修改

### 3.2 账号体系与登录流程

#### 学生账号
- **账号**：学号（8-12位数字）
- **初始密码**：`123456`
- **首次登录**：强制要求修改密码（密码强度：至少6位）
- **账号来源**：管理员通过 Excel 批量导入生成

#### 管理员/工作人员账号
- **账号**：学号或工号
- **初始密码**：`123456`
- **首次登录**：强制修改密码
- **账号来源**：种子数据预设（初始管理员）+ 后台创建

#### 登录流程
```
1. 用户输入学号/工号 + 密码
2. 系统验证账号密码
3. 如果是首次登录（isFirstLogin = true）：
   → 跳转到强制修改密码页面
   → 输入新密码（至少6位）
   → 确认新密码
   → 修改成功，isFirstLogin = false
   → 跳转到对应角色的首页
4. 如果不是首次登录：
   → 直接跳转到对应角色的首页
```

### 3.3 学生报名流程（导入 + 确认模式）

```
阶段一：管理员导入
  1. 管理员上传 Excel 表格（包含班委信息：姓名、学号、班级、职务等）
  2. 系统解析表格，自动创建学生账号（密码 123456）
  3. 系统识别职务信息，标记主职务和兼任职务
  4. 导入完成后，生成待确认名单

阶段二：学生确认报名
  1. 学生使用学号 + 初始密码 123456 登录
  2. 首次登录强制修改密码
  3. 进入"确认报名"页面，查看预填的信息（姓名、班级、学号、职务）
  4. 学生补充/确认以下信息：
     - 手机号（必填，如果表格中没有）
     - 邮箱（选填）
     - 第一场 / 第二场 选择（必填）
     - 备注（选填，如特殊情况说明）
  5. 点击"确认报名"，状态变为 PENDING（待审核）

阶段三：管理员审核
  1. 管理员在后台查看待审核报名
  2. 审核通过 → APPROVED
  3. 审核拒绝 → REJECTED（可填写拒绝原因）

阶段四：普通同学补位报名（可选）
  1. 如果某班班委有事或不足8人，开放普通同学自愿报名
  2. 普通同学通过"报名"页面注册账号并报名
  3. 注册时需填写：姓名、学号、班级、手机号、邮箱、第一场/第二场
  4. 普通同学的职务自动标记为"无"
  5. 同样需要管理员审核

阶段五：自动分组
  1. 管理员点击"自动分组"
  2. 系统按姓氏首字母排序
  3. 按顺序分为8-9人一队
  4. 自动识别每队队长（班长优先）
  5. 管理员可手动调整
  6. 点击"发布"，学生才能看到自己所在队伍

### 4.2 Schema 设计

```prisma
// 角色枚举（系统权限角色）
enum Role {
  TEACHER          // 老师
  ADMIN            // 活动管理员
  STAFF            // 工作人员
  STUDENT          // 学生（队长通过 TeamMember.isCaptain 标识）
}

// 学生班委职务
enum StudentPosition {
  CLASS_MONITOR    // 班长
  LEAGUE_SECRETARY // 团支书
  STUDY_COMMISSAR  // 学习委员
  LIFE_COMMISSAR   // 生活委员
  CULTURE_COMMISSAR // 文体委员
  NONE             // 无（普通同学补位报名）
}

// 管理员权限
enum Permission {
  MANAGE_REGISTRATIONS   // 管理报名
  MANAGE_TEAMS           // 管理分组
  MANAGE_STAFF           // 管理工作人员
  MANAGE_SCHEDULE        // 管理日程
  MANAGE_ANNOUNCEMENTS   // 管理公告
  MANAGE_SCORES          // 管理积分
  MANAGE_MATERIALS       // 管理物资
  MANAGE_ROTATION        // 管理轮转排班
  MANAGE_TREASURE        // 管理寻宝
  VIEW_FEEDBACKS         // 查看反馈
  MANAGE_SETTINGS        // 系统设置
  VIEW_LOGS              // 查看活动日志
  EXPORT_DATA            // 导出数据
  MANAGE_ADMINS          // 管理管理员（分配权限）
}

// 场次枚举
enum Session {
  FIRST            // 第一场
  SECOND           // 第二场
}

// 报名状态
enum RegistrationStatus {
  PENDING
  APPROVED
  REJECTED
}

// 工作人员申请状态
enum StaffApplicationStatus {
  PENDING
  APPROVED
  REJECTED
}

// 公告类型
enum AnnouncementType {
  NORMAL
  URGENT
  REMINDER
}

// 物资状态
enum MaterialStatus {
  PENDING
  PURCHASED
  ALLOCATED
  USED
}

// 游戏站
enum GameStation {
  LISTEN_COMMAND   // 听我口令
  DODGEBALL        // 躲避球
  CODE_BREAK       // 密码破译
  NO_TOUCH_GROUND  // 别碰地面
  TREASURE_HUNT    // 寻宝赛
}

// 线索卡等级
enum ClueCardTier {
  A                // 最优线索（第一轮积分高者）
  B                // 中等线索
  C                // 基础线索
}

// 用户
model User {
  id              String    @id @default(cuid())
  name            String    @db.VarChar(50)
  studentId       String    @unique @db.VarChar(20)  // 学号/工号，登录账号
  grade           Int?      // 2024 或 2025（管理员可能没有）
  className       String?   @db.VarChar(50)  // 班级（管理员可能没有）
  role            Role      @default(STUDENT)
  phone           String    @db.VarChar(20)
  email           String?   @db.VarChar(100)
  password        String    @db.VarChar(255)
  isFirstLogin    Boolean   @default(true)  // 是否首次登录（强制改密码）
  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt

  // 学生相关
  registration    Registration?
  teamMembers     TeamMember[]

  // 工作人员相关
  staffAssignment StaffAssignment?

  // 管理员/老师相关
  permissions     AdminPermission[]  // 管理员权限（仅 ADMIN 角色使用）
  scoresRecorded  Score[]   @relation("ScoreRecorder")
  feedbacks       Feedback[]
  albums          Album[]

### 7.3 关键功能说明

#### 学生导入（新增核心功能）
- 上传 Excel 文件（支持 .xlsx 格式）
- 系统自动解析表格，识别列：姓名、学号、班级、职务等
- **职务智能识别**：
  - 解析单元格文本，如 "班长兼团支书" → 主职务=班长，兼职=团支书
  - "班长兼学习委员" → 主职务=班长，兼职=学习委员
  - "团支书" → 主职务=团支书，无兼职
  - 关键词匹配优先级：班长 > 团支书 > 学习委员 > 生活委员 > 文体委员
- 预览导入结果，显示成功/失败行数
- 失败行显示具体原因（如学号重复、格式错误等）

## 十一、风险与注意事项

| 风险 | 影响 | 应对 |
